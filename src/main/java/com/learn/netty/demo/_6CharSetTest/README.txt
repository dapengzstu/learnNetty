字符编解码的问题：
1.  ASCLL
    7位表示一个字符，一共可以表示128种字符
2.  ISO-8859-1，兼容ascll
    8位，一个字节表示一个字符，
3.  GB2312，中国指定的，两个字节表示一个汉字
4.  GBK 是对GB2312的扩展
5.  GB18030，再次的扩展，最完整的表示简体中文
6.  big5    台湾对于繁体字的编码，和GB没关系

7.  unicode，最后，大统一了，表示了全世界的字符
    但是呢，unicode的存储容量有点大
8.  utf-8，utf-16，utf-32
    Unicode Translation Format

9.  utf和unicode有什么联系呢？
unicode是一种（编码方式），和Ascll是一个概念，比如 AB CD 这两字节表示某一个字符
utf是一种（存储方式），
utf-8 是unicode的一种实现方式
对于单字节的符号，比如英文，字节第一位设为0，后面7位为这个符号的unicode编码
对于n字节的符号，1110xxxx   10xxxxxx    10xxxxxx (10xxxxxx) 第一个字节前面有几个1就表示这个字符一共占用了几个字节，后面的字节都是以10开头的其他地方不全2

10. utf-16 也是两个字节表示，包括 utf-16LE,utf-16BE，小端和大端和计算机的存储结构有关系
计算机的文件，在文件的开头会有一个字符，0xFeFF（大端），0xFFFe（小端）表示某一个文件是大端还是小端的

11. utf-8变长的字节表示形式
如果是英文的那就是一个字节表示，如果是中文的就用三个字节表示，最长可以用6个字节表示一个字符

12. BOM（Byte Order Mark)字节序标记
比如0xFEFF就是一个种字节序标记，在文件的开头，是一种windows历史的遗留问题
